<h3>Introduction</h3>
	Object detection methods before yolo were using classifier for particular objects and look for that object at various locations and scales in image. Deformable Parts model uses a sliding window where classifier is run at evenly spaced locations over image. Recent methods like RCNN like generate bounding boxes and run classifier for these boxes. Then postprocessing is done to refine the bounding boxes, eliminate duplicates and rescore the boxes based on objects. 
	<p>
		RCNN is hard to optimize because each component must be trained separately. We need a single unified system that can be easily optimized. This is where YOLO comes into picture.  YOLO- single convolutional network that predicts bounding boxes and class scores for those boxes. Advantages of YOLO over traditional object detection are as follows
	</p>
	<ol>
		<li>Extremely fast - Since it is single convolutional network. </li>
		<li>Unlike sliding window and RCNN methods yolo see the entire image once and can encode contextual information.</li>
		<li>Yolo learns generalizable representations of images. When trained on natural images and tested on new images yolo outperforms the other detection methods. </li>
	</ol>
</div>

<div>
<h3>Unified Detection</h3>
	<p>
	Yolo unifies the components of object detection into a single neural network. So YOLO uses features from entire image to predict each bounding box. Yolo predicts all bounding boxes across all classes simultaneously. This makes YOLO possible for end to end training and real time speed with high average precision.	
	</p>
<h3>Convolutional Neural Network</h3>
	<p>
		Yolo uses series of convolutional layers followed by batch normalization and Leakyrelu  for feature extraction. Convolution with stride of 2 is used to downsample the image and no pooling is used. This helps in prevention of low level features often attribtued to pooling. Yolov3 uses darknet-53 for feature extraction which is shown in Fig 1. Since yolo has only series of convolutions, yolo can work with any input size. But ideally it is better to constant image size for other reasons. 
	</p>	
	<p>
		Yolo Network downsamples image by a factor called stride of the network. For yolov3. if we use the input of size (416, 416) then output will be of size (13,13). so stride of the network for yolov3 is 32.  
	</p>

<h3>Yolo Output</h3>
	<p>Each image is divided into S*S grid. If the center of object falls in grid, then grid is responsible for predicting that object. Each grid predicts B bounding boxes and confident scores of those boxes. Feature map in yolo will have B*(5+C) values where B bounding boxes specialise in detecting a certain kind of objects. For each bounding box we have (5+C) values that contains the  objectness score, bounding box coordinates  and confidence score for each of the classes. Confidence of object (objectness score) is defined as the probability of object multiplied by Intersection over union (IOU) between object and ground truth. 
	</p>
	<p>
		For example - Let us say image is divided into (13,13) grid and yolov3 predicts 3 bounding boxes for each grid. Then the cell containing the center of ground truth is responsible for predicting that object. Cell here is a point on the feature map. Output of feature map will be (13, 13, 3, 85) for 80 class output. If we flatten last two dimensions then shape will be (13, 13, 255). 
	</p>
	<p>
		Now for each cell we compute element wise product and extract probability that box contains an certain class. For each of the cells find maximum of probability scores across 3 anchor boxes and all classes. In practice yolo predicts too many boxes for a single object so we would like to filter that to smaller number of objects. We use Non Maximal Supression to do the same.  
	</p>
<h4>Anchor Boxes</h4>
	<p>
		Intuitively it makes sense to predict center, width and height of bounding box but in practice it leads to unstable gradients during training. Most of the recent object detectors predict the offset from the anchor boxes. These anchors are calculated using Kmeans clustering on the dataset. Yolo tries to learn how to predict these offsets. This works because our ground truth box should look like the anchor box we select and only little adjustment is needed, which gives a good start in training.
	</p> 
	<p>
			We can represent bounding box in many ways. Two of them are centroid representation and corner representation. In centroid representation we represent the bounding box as \( (x, y, w, h) \) and in corner representation we represent as \( (x_{min}, y_{min}, x_{max}, y_{max}) \).
	</p>

			Yolo predicts the offset that are relative to the top left corner of the grid cell and normalised by the dimensions of the cell from feature map (13). \(t_x\) and \(t_y\) should be less than 1 because if its greater than 1 center lies on the right of current cell. We take \( e^{t_w}\) because \(t_w\) can be negative number but width of the bounding box cannot be negative.

<h3>Multiple Scales</h3>
	<p>
		Yolov3 predicts objects using feature maps at different scales. Detections happen at feature maps of size - (13, 13), (26, 26), (52, 52). The network downsamples the input image until the first detection layer, where a detection is made using feature maps of a layer with stride 32. Further, layers are upsampled by a factor of 2 and concatenated with feature maps of a previous layers having identical feature map sizes. Another detection is now made at layer with stride 16. The same upsampling procedure is repeated, and a final detection is made at the layer of stride 8.
	</p>
	<p>
		At each scale,  each cell uses 3 anchors to predict 3 bounding boxes. This makes total number of anchors to 9 and these anchors are different for different scales. Multi scale helps in detecting smaller objects which was the problem in earlier versions of yolo. Upsampling can help the network learn fine grained features for detecting smaller objects. 
	</p>
<h3>Objectness Score and Confidence Scores</h3>
	<p>
		Objectness score tells us the probability of whether object is present in the class. It will be nearly 1 where the object is present else 0. Objectness score is passed through sigmoid to find probability. Class confidences represent the probability of detected objects belonging to each class. In earlier versions softmax was used but they have moved to sigmoid since softmax assumes classes are mutually exclusive.  
	</p>
<h3>Non Maximal Supression</h3>
<p>
	For an image of size (416, 416) yolo generates 10647 bounding boxes. So we need to filter these detections to number of objects present in an image. We remove the boxes that have low confidence (below certain threshold). Then we use NMS to remove multiple boxes that belong to same object. 
</p>
<p>
	NMS - Select the box that has highest score. Compute the overlap of the selected box with other boxes and then remove the boxes that have high IOU (greater than threshold). Iterate over until all boxes with lower confidence than the selected boxes are removed. After NMS only the best boxes with high confidence are remained. 
</p>
<h4>Training</h4>
	<p>
		In YOLO we have an object classifier and a bounding box regressor. Object classifier and bounding box regressor are trained jointly. Loss function used by Yolo is shown in Fig 2. Regressor is learning to predict the offset of the ground truth box from the anchor box. Yolo optimizes for sum squared error (easier to optimize) between the output of the network and ground truth boxes. 
	</p>
	<p>
	Sum squared error doesnot align with goal of maximising precision. Yolo weighs the localisation error and classification error equally which is not ideal. In every image grid cell doesnot contain objects, this can push confidence of those cell towards zero, often  overpowering  the  gradient  from  cells that do contain objects.  This can lead to model instability.
	</p>
	<p>
	So loss from bounding box predictions is increased and 	decrease the loss from confidence predictions for boxes that dont contain objects. 	Sum squared error weights large sized boxes and small sized boxes equally. To address this square root of weight and height are predicted from the network. Yolo penalises only classification error for the boxes that contain an object within  themselves it doesnot penalize empty boxes. 
	</p>
	<p>
		YOLO predicts multiple bounding boxes for each grid cell. But ideally we want only one bounding box to be responsible for the object. The bounding box that has greatest IOU with the ground truth is considered. This leads to specialization between the bounding box predictors.  Each bounding box gets better at predicting certain sizes, aspect ratios, or classes of object, improving overall recall
	</p>

<h4>Limitations</h4>
	<ol>
		<li>Difficult to generalise to objects in new aspect ratios.</li>
		<li>YOLO uses relatively coarse features for predicting bounding boxes since architecture has multiple downsampling layers from the input image</li>
		<li>Loss function treats differnce in large and small bounding boxes equally. </li>
		<li>Incorrect localisations</li>
	</ol>
