<h3>Introduction</h3>
	Object detection methods before yolo were using classifier for particular objects and look for that object at various locations and scales in image. Deformable Parts model uses a sliding window where classifier is run at evenly spaced locations over image. Recent methods like RCNN like generate bounding boxes and run classifier for these boxes. Then postprocessing is done to refine the bounding boxes, eliminate duplicates and rescore the boxes based on objects. 
	<p>
		RCNN is hard to optimize because each component must be trained separately. We need a single unified system that can be easily optimized. This is where YOLO comes into picture.  YOLO- single convolutional network that predicts bounding boxes and class scores for those boxes. Advantages of YOLO over traditional object detection are as follows
	</p>
	<ol>
		<li>Extremenly fast. No complex pipeline (it is single convolutional network) </li>
		<li>Unllike sliding window and RCNN methods yolo see the entire image once and can encode contextual information</li>
		<li>Yolo learns generalisable representations of images. When trained on natural images and tested on new images yolo outperforms the other detection methods. </li>
	</ol>
</div>

<h3>Unified Detection</h3>
	<p>
	Yolo unifies the components of object detection into a single neural network. So YOLO uses features from entire image to predict each bounding box. Yolo predicts all bounding boxes across all classes simultaneously. This makes YOLO possible for end to end training and real time speed with high average precision.	
	</p>
<h4>Network Design</h4>	
	<p>Each images is divided S*S grid. If the center of object falls in grid, then grid is responsible for predicting that object. Each grid predicts B bounding boxes and confidenct scores of those boxes. Confidence of object is defined as the probability of object multiplied by Intersection over union (IOU) between object and ground truth. Each grid cell also predicts C class conditional probability (probability of class given object). 	
	<img src="/assets/yolo_network.png" style="width:800px;height:400px;">
	</p>
<h4>Training</h4>
	<p>
		In YOLO we have an object classifier and a bounding box regressor. Bounding box consists of four parameters. They can be top-left and bottom right points or they could be center cordinates along with width and height of a bounding box. Now a days bounding boxes are set of four parameters encoded in a different way. 
	</p>
	<p>
		Object detection methods have a concept of anchor boxes or prior boxes. These are the predefined bounding boxes on input image. Bounding box regressor instead of predicting the bounding box it predicts the offset of the ground truth box from the anchor box. Regressor is learning to predict this offset from the anchor box. We optimize for sum squared error (easier to optimize) between the output of the network and ground truth boxes. 
	</p>
	<p>
	Sum squared error doesnot align with goal of maximising precision. Yolo weighs the localisation error and classification error equally which is not ideal. In every image grid cell doesnot contain objects, this can push confidence of those cell towards zero, often  overpowering  the  gradient  from  cells that do contain objects.  This can lead to model instability.
	</p>
	<p>
	So loss from bounding box predictions is increased and 	decrease the loss from confidence predictions for boxes that dont contain objects. 	Sum squared error weights large sized boxes and small sized boxes equally. To address this square root of weight and height are predicted from the network.
	</p>
	<p>
		YOLO predicts multiple bounding boxes for each grid cell. But ideally we want only one to be responsible for the object. The bounding box that has greatest IOU with the ground truth is considered. This leads to specialization between the bounding box predictors.  Each predictor gets better at predicting certainsizes, aspect ratios, or classes of object, improving overallrecall
	</p>

<h4>Inference</h4>
	<p>
	YOLO is fast at test time since it only requires a single network evaluation, unlike classifier-based methods. large objects can be localised by multiple grid cells. Non maximal suppression can be used to fix these multiple detections
</p>
<h4>Limitations</h4>
	<ol>
		<li>Difficult to generalise to objects in new aspect ratios.</li>
		<li>YOLO uses relatively coarse features for predicting bounding boxes since architecture has multiple downsampling layers from the input image</li>
		<li>Loss function treates differnce in large and small bounding boxes equally. </li>
		<li>Incorrect localisations</li>
	</ol>
